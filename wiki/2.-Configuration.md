# Configuration

## Plugin Init Options

```typescript
ClinkPlugin.init({
  relays: ['wss://relay.shocknetwork.com'],  // Required: Nostr relay URLs
  autoSettle: true,                           // Auto-confirm payments (default: true)
  offerExpiryMinutes: 30,                     // Payment window (default: 30)
  httpFallback: true,                         // Enable HTTP webhook fallback (default: true)
  nostrSecretKey: '',                         // Global Nostr key (optional, per-channel by default)
});
```

### Options Reference

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `relays` | `string[]` | `['wss://relay.shocknetwork.com']` | Nostr relay URLs for CLINK protocol communication |
| `autoSettle` | `boolean` | `true` | Automatically mark payments as settled when confirmed via Nostr |
| `offerExpiryMinutes` | `number` | `30` | Minutes before a payment offer expires |
| `httpFallback` | `boolean` | `true` | Enable HTTP POST webhook endpoint at `/payments/clink` |
| `nostrSecretKey` | `string` | `''` | Global Nostr secret key (hex). If empty, per-channel keys are generated |

## Per-Channel Configuration

Each Vendure channel gets its own Nostr keypair and relay configuration. On first access, a keypair is auto-generated and stored in the `clink_channel_config` table.

### Admin API — Query Channel Config

```graphql
query {
  clinkConfig(channelId: "1") {
    id
    nostrPubkey
    relayUrls
    autoSettle
    offerExpiryMinutes
    httpFallback
    paymentHandlerCode
  }
}
```

### Admin API — Update Channel Config

```graphql
mutation {
  updateClinkConfig(
    channelId: "1",
    input: {
      relayUrls: ["wss://relay.shocknetwork.com", "wss://relay2.example.com"]
      autoSettle: true
      offerExpiryMinutes: 60
      httpFallback: true
    }
  ) {
    id
    nostrPubkey
    relayUrls
  }
}
```

## Payment Handler Args

The `clink-lightning` payment method accepts these args when added to an order:

| Arg | Type | Default | Description |
|-----|------|---------|-------------|
| `relayUrls` | `string` | `wss://relay.shocknetwork.com` | Comma-separated relay URLs |
| `offerExpiryMinutes` | `int` | `30` | Minutes until offer expires |

## Environment Variables

For production, set sensitive values via environment variables:

```bash
# Optional: global Nostr secret key (overrides per-channel generation)
CLINK_NOSTR_SECRET_KEY=your_hex_secret_key

# Optional: relays (comma-separated)
CLINK_RELAYS=wss://relay.shocknetwork.com,wss://relay2.example.com
```
