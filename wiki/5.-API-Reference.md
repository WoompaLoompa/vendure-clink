# API Reference

## Shop API

### `clinkOffer`

Get the CLINK payment offer for an order.

```graphql
query ClinkOffer($orderCode: String!) {
  clinkOffer(orderCode: $orderCode) {
    id
    createdAt
    updatedAt
    offerId
    noffer
    amountSats
    pricingType
    status
    bolt11Invoice
    expiresAt
    paidAt
  }
}
```

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `orderCode` | `String!` | Yes | The order code to look up |

**Returns**: `ClinkOffer` or `null`

---

## Admin API

### `clinkConfig`

Get the CLINK configuration for a channel.

```graphql
query ClinkConfig($channelId: ID!) {
  clinkConfig(channelId: $channelId) {
    id
    createdAt
    updatedAt
    nostrPubkey
    relayUrls
    autoSettle
    offerExpiryMinutes
    httpFallback
    paymentHandlerCode
  }
}
```

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `channelId` | `ID!` | Yes | The channel ID |

**Returns**: `ClinkChannelConfig` or `null`

**Required permission**: `ReadAdministrator`

---

### `clinkOffers`

List all CLINK offers with optional filters.

```graphql
query ClinkOffers($status: String, $limit: Int, $offset: Int) {
  clinkOffers(status: $status, limit: $limit, offset: $offset) {
    totalItems
    items {
      id
      createdAt
      updatedAt
      offerId
      noffer
      amountSats
      pricingType
      status
      bolt11Invoice
      expiresAt
      paidAt
    }
  }
}
```

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `status` | `String` | No | Filter by status: `pending`, `paid`, `expired`, `cancelled` |
| `limit` | `Int` | No | Max results (default: 50) |
| `offset` | `Int` | No | Pagination offset |

**Returns**: `ClinkOfferList!`

**Required permission**: `ReadAdministrator`

---

### `updateClinkConfig`

Update CLINK configuration for a channel.

```graphql
mutation UpdateClinkConfig($channelId: ID!, $input: ClinkConfigInput!) {
  updateClinkConfig(channelId: $channelId, input: $input) {
    id
    nostrPubkey
    relayUrls
    autoSettle
    offerExpiryMinutes
    httpFallback
    paymentHandlerCode
  }
}
```

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `channelId` | `ID!` | Yes | The channel ID |
| `input` | `ClinkConfigInput!` | Yes | Configuration updates |

**`ClinkConfigInput`**:
| Field | Type | Description |
|-------|------|-------------|
| `relayUrls` | `[String!]` | Nostr relay URLs |
| `autoSettle` | `Boolean` | Auto-settle payments |
| `offerExpiryMinutes` | `Int` | Offer expiry in minutes |
| `httpFallback` | `Boolean` | Enable HTTP webhook |

**Required permission**: `ReadAdministrator`

## Types

### `ClinkOffer`

```graphql
type ClinkOffer implements Node {
  id: ID!
  createdAt: DateTime!
  updatedAt: DateTime!
  offerId: String!          # Unique offer identifier
  noffer: String!           # Bech32-encoded CLINK offer (noffer1...)
  amountSats: Int!          # Amount in satoshis
  pricingType: Int!         # 0=Fixed, 1=Range, 2=Custom
  status: String!           # pending | paid | expired | cancelled
  bolt11Invoice: String     # Lightning invoice (if resolved)
  expiresAt: DateTime       # Offer expiration time
  paidAt: DateTime          # Payment confirmation time
}
```

### `ClinkPaymentStatus`

```graphql
type ClinkPaymentStatus {
  offerId: String!
  status: String!
  bolt11Invoice: String
  preimage: String          # Payment preimage (proof of payment)
  noffer: String
  amountSats: Int!
}
```

### `ClinkChannelConfig`

```graphql
type ClinkChannelConfig implements Node {
  id: ID!
  createdAt: DateTime!
  updatedAt: DateTime!
  nostrPubkey: String!      # Channel's Nostr public key
  relayUrls: [String!]!     # Configured relay URLs
  autoSettle: Boolean!      # Auto-settlement enabled
  offerExpiryMinutes: Int!  # Offer expiry duration
  httpFallback: Boolean!    # HTTP webhook enabled
  paymentHandlerCode: String! # Payment method code
}
```

### `ClinkOfferList`

```graphql
type ClinkOfferList implements PaginatedList {
  items: [ClinkOffer!]!
  totalItems: Int!
}
```

## HTTP Webhook Endpoints

### `POST /payments/clink`

Direct payment notification (for wallets without Nostr support).

**Request**:
```json
{
  "offerId": "vendure_ORDERCODE_1234567890",
  "bolt11": "lnbc1...",
  "preimage": "hex_preimage_string",
  "amount_sats": 50000,
  "timestamp": 1234567890
}
```

**Response (success)**:
```json
{
  "success": true,
  "message": "Payment settled successfully"
}
```

**Response (error)**:
```json
{
  "success": false,
  "message": "Offer not found: invalid_offer_id"
}
```

### `POST /payments/clink/webhook`

Generic webhook endpoint for payment confirmations.

**Request**:
```json
{
  "type": "payment_confirmed",
  "data": {
    "offerId": "vendure_ORDERCODE_1234567890",
    "bolt11": "lnbc1...",
    "preimage": "hex_preimage_string"
  }
}
```

**Response**:
```json
{
  "received": true
}
```

## Payment Handler Code

The payment method is registered with code `clink-lightning`. Use this when adding payment to an order:

```graphql
mutation {
  addPaymentToOrder(
    input: {
      method: "clink-lightning"
      metadata: {}
    }
  ) {
    ... on Order {
      id
      code
      total
      state
    }
    ... on ErrorResult {
      errorCode
      message
    }
  }
}
```
